# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.1.0] - 2026-04-26

### Added
- **`adr-scribe` Skill**: A new skill to draft, retro-document, and review Architecture Decision Records (ADRs).
- **`example-codebase` Sandbox**: A mock React/Node.js/Postgres application to test AI agents safely without exposing proprietary code.
- **Behavioral Testing**: `validate-c4-diagram.sh` script to parse and validate Mermaid C4 syntax generated by agents.
- **Cross-Skill Integration**: Agents using `c4designer` are now prompted to use `adr-scribe` when structural changes occur.
- **Community Standards**: Added `CODE_OF_CONDUCT.md` and `CONTRIBUTING.md`.
- **Infrastructure**: Added `LICENSE` (MIT) and `.gitignore`.

### Changed
- Refactored `test-skill-format.sh` to dynamically loop over all skills in the repository.
- Re-wrote `README.md` to be more accessible to non-technical users and to document the new `adr-scribe` skill.

## [1.0.0] - Initial Release

### Added
- Created the universal `c4-skills` repository.
- Agent-agnostic `c4designer` skill structure designed to be easily copy-pasted across different coding agents.
- **5 core modes**:
  - `mode-design.md`: Greenfield architecture dialogue.
  - `mode-document-code.md`: Retro-documentation from scanning existing codebases.
  - `mode-document-prose.md`: Retro-documentation from a textual source.
  - `mode-review.md`: Automated critique against Simon Brown's C4 model checklist.
  - `mode-update.md`: Incremental updates to existing C4 diagrams.
- **Reference material**: `mermaid-c4-syntax.md`, `review-checklist.md`, `level-template.md`, and `supporting-diagrams.md`.
- Automated test script `test-skill-format.sh` to validate the skill structure and formatting.
